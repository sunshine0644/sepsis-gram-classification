Model Specifications

Model Type LightGBM (Gradient Boosting)
Algorithm Gradient Boosting Decision Trees
Input Features 14 clinical indicators × 3 timepoints
Output Binary classification (Gram-positive/Gram-negative)

Training Data

Source MIMIC-III Clinical Database
Patient Population Sepsis patients with bloodstream infections
Time Coverage First 24 hours of ICU admission
Data Points 3 timepoints per patient (0-8h, 8-16h, 16-24h)

Clinical Features

  • Vital Signs: Heart rate, Blood pressure, Respiratory rate, Oxygen saturation
  • Hematology: White blood cells, Hemoglobin, Platelets
  • Biochemistry: Blood urea nitrogen, Glucose
  • Coagulation: Prothrombin time
  • Electrolytes: Sodium, Potassium, Chloride, Bicarbonate

Important Information

Key considerations:

  • Not a substitute for clinical expertise

Clinical Guidelines

How to Use This Tool

  1. Enter patient data from three timepoints (ideally 0-8h, 8-16h, 16-24h)
  2. Ensure data quality - accurate measurements lead to better predictions
  3. Review prediction results along with confidence levels
  4. Consider the clinical advice provided with each prediction
  5. Always verify predictions with other clinical findings

Clinical Workflow Integration

  • Use as part of comprehensive sepsis evaluation
  • Combine with microbiology results when available
  • Consider patient-specific factors and comorbidities
  • Review antibiotic therapy in light of predictions
  • Document use of decision support in patient records

Professional Responsibility

  • You remain responsible for all clinical decisions
  • This tool does not replace clinical expertise
  • Report any discrepancies between predictions and clinical findings
  • Provide feedback to improve the system
